# Руководство по развертыванию MerAI & Monitoring

## Обзор проекта

MerAI & Monitoring представляет собой полностью переработанную систему мониторинга Telegram чатов с уникальной архитектурой Dual Mode. Система реализует все требования технического задания с использованием Python в качестве основного языка программирования, что обеспечивает стабильную работу как с Telegram Bot API, так и с Userbot функциональностью через библиотеку Telethon.

## Ключевые изменения относительно предыдущей версии

Предыдущая версия проекта содержала множество функций, которые были удалены в соответствии с вашими требованиями. Полностью исключены система уровней и опыта XP, все достижения и бейджи, умный поиск с категоризацией, коллекции и закладки сообщений, система тегов, скам детектор, триггеры на ключевые слова, тепловые карты активности, расширенная аналитика и медиа галерея. Вместо этого реализован фокусированный функционал с Dual Mode мониторингом, детектором контактов, подписочной моделью и нативными чеклистами.

## Технологический стек

Проект использует Python версии 3.11 или выше как основной язык разработки. Для работы с Telegram Bot API применяется библиотека aiogram версии 3.13, которая обеспечивает полную поддержку Business API и современных функций Telegram. Режим Userbot реализован с использованием Telethon версии 1.37, выбранного за его стабильную работу с новыми ограничениями авторизации Telegram, где код приходит в сообщении Telegram вместо SMS. База данных построена на SQLite3 с возможностью миграции на PostgreSQL при необходимости масштабирования. Все медиафайлы сохраняются в файловой системе с организацией по директориям пользователей.

## Структура проекта

Корневая директория содержит файл config.py с централизованной конфигурацией всех параметров системы, включая настройки подписок, брендинга и ограничений API. Модуль database.py реализует все операции с базой данных через объектно-ориентированный интерфейс. Файл main.py служит точкой входа в приложение и координирует работу всех компонентов.

Директория bot содержит основные обработчики команд и событий в файле handlers.py, дополнительные функции для чеклистов и детектора контактов в additional_handlers.py, генераторы клавиатур в keyboards.py и вспомогательные утилиты в utils.py. Модуль userbot включает manager.py с полной реализацией Telethon клиента для мониторинга через userbot режим. Админ панель размещена в директории admin с handlers.py, обеспечивающим управление пользователями и системой.

## Dual Mode Architecture

Система поддерживает три режима работы одновременно. Режим Business API работает с бизнес-чатами и требует наличия Telegram Premium на аккаунте пользователя. Этот режим имеет ограничения по сохранению сообщений с таймером самоуничтожения и лимитом на просмотр. Режим Userbot функционирует через личный аккаунт Telegram пользователя и может отслеживать все типы чатов кроме секретных. Dual Mode комбинирует оба подхода для максимального покрытия всех сценариев использования.

Переключение между режимами осуществляется через меню Dual Mode в интерфейсе бота. При выборе Userbot режима система запрашивает номер телефона и код подтверждения, который приходит в Telegram. Сессия сохраняется в зашифрованном виде и автоматически восстанавливается при перезапуске системы.

## Подписочная модель

Система монетизации построена на Telegram Stars с прямым доступом к балансу пользователя для автоматического продления подписок. Новые пользователи получают трехдневный пробный период при первом подключении Business API или активации Userbot режима.

Базовый тариф предоставляет мониторинг до 1000 сообщений в месяц за 50 Stars, что эквивалентно примерно 100 рублям. Включены функции экспорта данных и базового поиска по сообщениям. Премиум тариф за 150 Stars обеспечивает безлимитный мониторинг, полный доступ к Dual Mode, детектору контактов и чеклистам. Lifetime подписка за 500 Stars предоставляет пожизненный доступ ко всем функциям включая эксклюзивные возможности и VIP поддержку.

Автопродление подписки настраивается пользователем через интерфейс бота. При наличии достаточного баланса Stars система автоматически продлевает подписку за день до окончания текущего периода. Все транзакции логируются в базе данных для административной отчетности.

## Бонусная система

Пользователи получают семь дополнительных дней подписки при подключении собственного бота к системе. Процесс начинается с создания нового бота через BotFather и получения токена. Пользователь отправляет токен в интерфейсе MerAI & Monitoring, система верифицирует токен через Telegram API и автоматически продлевает подписку на семь дней. Подключенные боты сохраняются в базе данных и могут использоваться для расширения функциональности в будущем.

## Детектор контактов

Система автоматически анализирует каждое сохраненное сообщение на наличие контактной информации. Регулярные выражения детектируют телефонные номера в международных форматах, email-адреса согласно RFC стандартам, Telegram username в формате @username и URL-ссылки с валидацией протокола.

Обнаруженные контакты сохраняются в отдельной таблице базы данных с привязкой к исходному сообщению и чату. Пользователи могут просматривать контакты с фильтрацией по типу, экспортировать список контактов в различных форматах и получать статистику по каждому типу контактной информации.

## Чеклисты

Реализованы нативные чеклисты с интеграцией в Telegram интерфейс. Пользователи создают чеклисты с произвольным названием, добавляют неограниченное количество элементов до 200 символов каждый и управляют состоянием через inline кнопки. Система отображает прогресс выполнения в процентах и сохраняет историю изменений для каждого чеклиста.

## Административная панель

Админ панель доступна только пользователю с ID, указанным в ADMIN_ID конфигурации. Функционал включает просмотр всех пользователей с детальной статистикой, ручную выдачу любого тарифа подписки без оплаты, блокировку и разблокировку аккаунтов, очистку истории диалогов с пользователями и просмотр всех транзакций с группировкой по тарифам.

Система логирования сохраняет все административные действия в базе данных с отметками времени и описанием операций. Статистика обновляется в реальном времени и включает активных пользователей за различные периоды, распределение подписок по тарифам и общую выручку в Stars и рублевом эквиваленте.

## Управление брендингом

Через админ панель доступна настройка текстовых элементов интерфейса. Приветственное сообщение изменяется через команду в интерфейсе с сохранением в базе данных. Цветовая схема настраивается в config.py с возможностью переопределения через базу данных. Footer текст добавляется ко всем сообщениям бота автоматически.

## Развертывание на Production

Для работы на production сервере создайте конфигурационный файл .env на основе .env.example с вашими реальными токенами и ключами. Установите все зависимости из requirements.txt с параметром break-system-packages если используется системный Python. Настройте systemd service для автоматического запуска и перезапуска бота при сбоях. Реализуйте регулярное резервное копирование файла database/merai.db для защиты данных пользователей. Настройте ротацию логов для предотвращения переполнения диска.

## Развертывание на BotHost

Платформа BotHost поддерживает автоматическое развертывание Python проектов. Создайте новый проект и загрузите файлы через git репозиторий или файловый менеджер. Выберите Python 3.11 как runtime environment в настройках проекта. Установите зависимости через встроенный менеджер пакетов, указав requirements.txt. Настройте переменные окружения через веб-интерфейс без редактирования файлов. Установите main.py как точку входа и запустите проект через кнопку Deploy.

## Тестирование функциональности

После развертывания протестируйте основные функции системы. Отправьте команду /start вашему боту для проверки базовой работоспособности и регистрации в системе. Подключите Business API через настройки Telegram Business Tools для активации пробного периода. Настройте Userbot режим через меню Dual Mode с вводом номера телефона и кода подтверждения.

Проверьте мониторинг удаленных сообщений отправив и удалив тестовое сообщение в чате. Убедитесь в сохранении отредактированных сообщений с форматом «Было/Стало». Протестируйте детектор контактов отправив сообщение с телефоном, email и ссылками. Создайте тестовый чеклист для проверки функциональности управления задачами.

## Решение возможных проблем

При ошибках импорта Telethon убедитесь в установке всех зависимостей из requirements.txt без пропусков. Проверьте версию Python командой python3 --version, минимальная требуемая версия 3.11. Если база данных не создается автоматически проверьте права доступа к директории database и наличие достаточного места на диске.

При проблемах авторизации Userbot убедитесь в корректности API ID и API Hash из my.telegram.org. Проверьте формат номера телефона с обязательным префиксом плюс и кодом страны. Запросите новый код если предыдущий истек через пять минут. При сохранении сообщений проверьте наличие активной подписки через меню Статистика и убедитесь в непревышении лимита сообщений для текущего тарифа.

## Мониторинг и поддержка

Система ведет детальное логирование всех операций в файл logs/merai.log. Критические ошибки сопровождаются полным стектрейсом для упрощения отладки. Действия пользователей сохраняются в таблицу action_logs базы данных для аудита и анализа. Административные операции логируются отдельно с указанием ID администратора и временных меток.

Настройте мониторинг процесса через supervisor или systemd для автоматического перезапуска при сбоях. Реализуйте уведомления администратора при критических ошибках через отдельного Telegram бота или email систему. Регулярно проверяйте размер базы данных и логов для предотвращения проблем с дисковым пространством.

## Безопасность и конфиденциальность

Все чувствительные данные хранятся в переменных окружения или конфигурационном файле с ограниченным доступом. Токены ботов и API ключи никогда не передаются клиентской стороне или в логах. Userbot сессии сохраняются в зашифрованном виде через механизм StringSession библиотеки Telethon. Медиафайлы организованы по директориям пользователей с проверкой прав доступа при каждом запросе.

База данных может быть зашифрована дополнительно через SQLCipher при необходимости повышенной защиты. Рекомендуется использовать HTTPS для всех веб-интерфейсов и webhook соединений. Регулярно обновляйте зависимости для получения security патчей и исправлений уязвимостей.

## Масштабирование

При росте количества пользователей рассмотрите миграцию с SQLite на PostgreSQL для улучшения производительности. Реализуйте Redis для кэширования частых запросов к базе данных. Настройте load balancer для распределения нагрузки между несколькими инстансами бота. Используйте отдельные серверы для основного бота и userbot процессов для изоляции и надежности.

Медиафайлы можно хранить на S3-совместимом хранилище вместо локальной файловой системы. Реализуйте очередь задач через Celery или RQ для асинхронной обработки тяжелых операций. Настройте репликацию базы данных для обеспечения высокой доступности и отказоустойчивости.

## Заключение

Проект MerAI & Monitoring представляет собой полностью переработанную систему мониторинга с фокусом на ключевую функциональность и стабильную работу. Dual Mode архитектура обеспечивает максимальное покрытие различных сценариев использования. Подписочная модель через Telegram Stars упрощает монетизацию и управление платежами. Административная панель предоставляет полный контроль над системой и пользователями.

Все компоненты протестированы и готовы к production развертыванию. Документация включает подробные инструкции по установке, настройке и использованию системы. Техническая поддержка доступна через контакты администратора для решения возникающих вопросов и проблем.
