## 🚀 Возможности

MerAI & Monitoring представляет собой комплексное решение для мониторинга Telegram чатов с уникальной системой Dual Mode, позволяющей работать одновременно через Business API и Userbot.

### Основные функции

**Dual Mode Operation** - система поддерживает два режима работы одновременно. Режим Business API обеспечивает мониторинг бизнес-чатов с требованием Telegram Premium. Режим Userbot позволяет отслеживать все типы чатов, включая обычные диалоги. Комбинированный режим Dual обеспечивает максимальное покрытие всех возможных сценариев использования.

**Мониторинг сообщений** включает отслеживание удаленных сообщений с сохранением текста и медиа-файлов, фиксацию отредактированных сообщений с форматом «Было/Стало», и автоматическое создание ZIP-архивов при удалении пяти или более сообщений из одного чата.

**Детектор контактов** автоматически выявляет в сообщениях телефонные номера по международным форматам, email-адреса, Telegram-username в формате @username, и URL-ссылки.

**Подписочная модель** реализована через Telegram Stars с прямым доступом к балансу пользователя. Система включает пробный период продолжительностью три дня для новых пользователей, три основных тарифа (Базовый, Премиум, Lifetime), и автоматическое продление подписок при наличии средств.

**Бонусная система** предоставляет семь дополнительных дней подписки при подключении собственного бота пользователя. Процесс подключения осуществляется через токен от BotFather с автоматической верификацией.

**Административная панель** обеспечивает управление пользователями, включая блокировку и разблокировку, ручную выдачу подписок, просмотр детальной статистики по каждому пользователю, очистку истории диалогов, управление настройками брендинга, и массовую рассылку сообщений.

## 📋 Требования

Для работы системы необходимы следующие компоненты: Python версии 3.11 или выше, токен Telegram Bot от BotFather, Telegram API credentials (API ID и API Hash) для работы Userbot, SQLite3 для базы данных (встроен в Python), и сервер с постоянным подключением к интернету для хостинга.

## 🔧 Установка

### Шаг 1: Клонирование и подготовка

Создайте директорию проекта и перейдите в нее. Установите все необходимые зависимости из файла requirements.txt используя команду pip install с параметром break-system-packages для корректной установки в системном окружении.

### Шаг 2: Конфигурация

Откройте файл config.py и настройте основные параметры. Укажите ваш токен бота в переменной BOT_TOKEN, полученный от BotFather. Для работы Userbot необходимо получить API ID и API Hash на сайте my.telegram.org в разделе API development tools. Установите ваш Telegram ID в качестве администратора в переменной ADMIN_ID.

### Шаг 3: Инициализация базы данных

База данных SQLite создается автоматически при первом запуске системы. Файл базы данных располагается в директории database/merai.db. Все необходимые таблицы создаются автоматически модулем database.py при инициализации.

### Шаг 4: Запуск

Запустите основной файл main.py. При успешном запуске вы увидите приветственное сообщение с логотипом системы, информацию о подключенном боте и статистику базы данных. Система начнет работу в режиме polling и будет готова к приему сообщений.

## 🎯 Использование

### Для пользователей

Первый запуск начинается с отправки команды /start вашему боту. Система автоматически зарегистрирует вас в базе данных. При первом подключении Business API вы автоматически получите трехдневный пробный период.

Подключение Business API требует наличия Telegram Premium на вашем аккаунте. В настройках Telegram выберите раздел Business, добавьте бота в список бизнес-инструментов. После подключения бот начнет автоматически сохранять все сообщения из ваших бизнес-чатов.

Для работы через Userbot выберите соответствующий режим в настройках бота. Введите номер телефона, на который зарегистрирован ваш Telegram аккаунт. Код подтверждения придет в сообщении Telegram, а не через SMS. Введите полученный код для авторизации. После успешной авторизации Userbot начнет мониторинг всех ваших чатов.

Бонусная программа позволяет получить семь дополнительных дней подписки. Создайте нового бота через BotFather, получите его токен, нажмите кнопку «Получить бонус» в интерфейсе бота и отправьте токен созданного бота. После верификации бонусные дни будут автоматически добавлены к вашей подписке.

### Для администраторов

Доступ к админ панели осуществляется через кнопку «Админ Панель», видимую только пользователю с ID, указанным в ADMIN_ID. В панели доступен просмотр списка всех пользователей с фильтрацией по статусу подписки и активности.

Управление пользователями включает возможность блокировки и разблокировки аккаунтов, ручную выдачу любого тарифа подписки, просмотр детальной статистики по сообщениям пользователя, и очистку истории диалога с конкретным пользователем.

Статистика системы предоставляет информацию об общем количестве пользователей, количестве активных подписок с разбивкой по тарифам, новых пользователях за последнюю неделю, активных пользователях за последние сутки, и общем количестве сохраненных сообщений.

Финансовый мониторинг позволяет просматривать все транзакции с Telegram Stars, отслеживать общую выручку в Stars и рублевом эквиваленте, и анализировать популярность различных тарифов подписки.

## 📊 Архитектура проекта

Проект организован в модульную структуру для удобства поддержки и расширения.

Корневая директория содержит файл config.py с централизованной конфигурацией всех параметров системы, database.py, реализующий взаимодействие с SQLite базой данных, и main.py как точку входа в приложение с инициализацией всех компонентов.

Модуль bot включает handlers.py с основными обработчиками команд и событий, keyboards.py для генерации inline клавиатур, и utils.py с вспомогательными функциями и утилитами.

Модуль userbot содержит manager.py, реализующий работу с Telethon для мониторинга через userbot режим.

Модуль admin предоставляет handlers.py с обработчиками административной панели и управления системой.

Директория database хранит файл merai.db с SQLite базой данных.

Директория media используется для сохранения медиафайлов из сообщений с организацией по user_id и типу медиа.

Директория logs содержит файл merai.log с логами работы системы.

## 🔐 Безопасность

Система реализует несколько уровней защиты данных пользователей.

Токены и API ключи хранятся в конфигурационном файле и не должны передаваться третьим лицам. Рекомендуется использовать переменные окружения для production развертывания.

Базы данных SQLite обеспечивают локальное хранение всех данных с возможностью шифрования файла базы данных. Доступ к административным функциям строго ограничен проверкой ADMIN_ID.

Userbot сессии сохраняются в зашифрованном виде через механизм StringSession библиотеки Telethon. Сессии привязаны к конкретному устройству и пользователю.

Медиафайлы организованы по директориям пользователей с ограниченным доступом только для владельца. Система автоматически проверяет валидность подписки перед сохранением новых данных.

## 🚫 Ограничения Telegram API

При разработке учитывались следующие ограничения официального Telegram API.

Business API не может сохранять сообщения с таймером самоуничтожения, сообщения с лимитом на один просмотр, и сообщения из секретных чатов. При удалении целого чата Business API не получает соответствующее событие и не может создать архив автоматически.

Userbot через Telethon преодолевает некоторые из этих ограничений. Он может отслеживать удаление обычных чатов и создавать архивы, сохранять медиафайлы из удаленных сообщений до момента удаления, и работать со всеми типами чатов кроме секретных.

## 📝 Логирование

Система ведет детальное логирование всех операций. Основной лог-файл расположен в logs/merai.log. Уровень детализации логирования настраивается через параметр LOG_LEVEL в config.py. Критические ошибки логируются с полным стектрейсом для упрощения отладки.

Действия пользователей сохраняются в таблицу action_logs базы данных. Административные действия логируются отдельно с указанием администратора. Система автоматически ротирует лог-файлы при достижении определенного размера.

## 🔄 Обновления

Для обновления системы до новой версии выполните следующие действия. Сделайте резервную копию файла database/merai.db перед началом обновления. Остановите текущий процесс бота. Загрузите новую версию файлов проекта, сохранив файл config.py с вашими настройками. Обновите зависимости через pip install для файла requirements.txt. Запустите систему заново с проверкой логов на наличие ошибок.

## 🛠 Развертывание на Production

Для развертывания на production сервере рекомендуется использовать следующую конфигурацию.

Используйте systemd service для автоматического запуска и перезапуска бота. Настройте reverse proxy через nginx для веб-интерфейса если планируется его добавление. Реализуйте регулярное резервное копирование базы данных. Настройте мониторинг процесса через supervisor или аналогичные инструменты. Используйте переменные окружения для чувствительных данных вместо хардкода в config.py.

Для хостинга на платформе BotHost создайте новый проект, загрузите все файлы проекта через git или файловый менеджер, установите Python 3.11+ в настройках проекта, установите зависимости через requirements.txt, настройте переменные окружения для BOT_TOKEN и других параметров, и запустите проект через main.py.
